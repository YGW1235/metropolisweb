# Metropolis Admin Guide

Metropolis 운영자를 위한 관리자 가이드입니다.

이 문서는 토론 주제 운영, 공지사항 관리, 신고 처리, 유저 테스트, 운영 중 문제 대응 기준을 정리합니다.

## 1. 관리자 접속

관리자 계정으로 로그인한 뒤 아래 주소에 접속합니다.

```txt
/admin
```

관리자 권한이 없는 계정은 관리자 페이지에 접근할 수 없습니다.

관리자 권한은 Supabase `profiles.role` 값이 `admin`인 유저에게 부여됩니다.

```sql
update public.profiles
set role = 'admin'
where email = '관리자이메일@example.com';
```

## 2. 토론 주제 운영

관리자는 토론 주제를 생성하고 상태를 변경할 수 있습니다.

### 주제 생성

관리자 페이지에서 새 주제를 생성합니다.

필수 입력값:

```txt
제목
설명
상태
아테나 측 입장
포세이돈 측 입장
```

선택 입력값:

```txt
시작 예정 시간
종료 예정 시간
```

### 주제 상태

토론 주제는 다음 상태를 가집니다.

```txt
draft
```

관리자 준비 상태입니다. 일반 유저에게 보이지 않고 참가할 수 없습니다.

```txt
open
```

일반 유저에게 공개됩니다. 신규 참가가 가능하고, 참가자는 글과 댓글을 작성할 수 있습니다.

```txt
active
```

토론 진행 상태입니다. 기존 참가자는 글과 댓글을 작성할 수 있지만 신규 참가자는 막힙니다.

```txt
closed
```

종료된 토론입니다. 일반 유저가 읽을 수는 있지만 신규 참가, 글 작성, 댓글 작성은 불가능합니다.

```txt
archived
```

보관 상태입니다. 일반 유저에게 보이지 않습니다.

### 추천 운영 흐름

일반적인 토론 운영 흐름은 다음과 같습니다.

```txt
draft → open → active → closed → archived
```

권장 방식:

```txt
1. draft 상태로 주제 작성
2. 내용 검토 후 open으로 공개
3. 참가자가 충분히 모이면 active로 변경
4. 토론이 끝나면 closed로 변경
5. 오래된 주제는 archived로 보관
```

## 3. 참가자 운영

유저는 open 상태의 주제에 참가할 수 있습니다.

참가 시 시스템이 자동으로 찬성 또는 반대 진영을 배정합니다.

```txt
pro = 아테나 진영
con = 포세이돈 진영
```

참가자는 화면에서 실명이 아니라 익명 번호로 표시됩니다.

예시:

```txt
아테나 진영 익명 1
포세이돈 진영 익명 2
```

운영자는 특정 유저의 실제 이메일이나 UUID를 공개 화면에 노출하지 않도록 주의해야 합니다.

## 4. 게시글 운영

참가자는 토론방에서 게시글을 작성할 수 있습니다.

게시글에는 다음 정보가 포함됩니다.

```txt
제목
내용
진영
작성 시간
이미지
익명 작성자 라벨
```

게시글 이미지는 Supabase Storage의 `debate-images` 버킷에 저장됩니다.

허용 이미지 조건:

```txt
JPG
PNG
WEBP
최대 5MB
```

### 게시글 삭제

작성자는 본인 게시글을 삭제할 수 있습니다.

삭제된 게시글은 실제 DB row가 바로 제거되는 것이 아니라 `status = deleted` 방식으로 처리됩니다.

관리자는 신고 처리 과정에서 게시글을 숨길 수 있습니다.

```txt
visible = 공개
hidden = 관리자에 의해 숨김
deleted = 작성자 삭제
```

## 5. 댓글 운영

참가자는 게시글 상세 페이지에서 댓글을 작성할 수 있습니다.

댓글도 작성자의 실제 계정 정보 대신 익명 라벨로 표시됩니다.

댓글 상태:

```txt
visible = 공개
hidden = 관리자에 의해 숨김
deleted = 작성자 삭제
```

작성자는 본인 댓글을 삭제할 수 있습니다.

관리자는 신고 처리 과정에서 댓글을 숨길 수 있습니다.

## 6. 신고 처리

유저는 게시글 또는 댓글을 신고할 수 있습니다.

신고 대상:

```txt
post
comment
```

신고 상태:

```txt
pending
reviewing
resolved
dismissed
```

상태 의미:

```txt
pending
```

신규 신고입니다. 아직 관리자가 처리하지 않았습니다.

```txt
reviewing
```

관리자가 확인 중인 신고입니다.

```txt
resolved
```

관리자가 신고를 받아들여 대상 게시글 또는 댓글을 숨김 처리한 상태입니다.

```txt
dismissed
```

관리자가 신고를 기각한 상태입니다.

### 신고 처리 기준

신고가 들어오면 다음 순서로 확인합니다.

```txt
1. 신고 대상이 게시글인지 댓글인지 확인
2. 신고 사유 확인
3. 실제 내용 확인
4. 토론 맥락상 허용 가능한 표현인지 판단
5. 처리 방식 선택
```

처리 방식:

```txt
검토 중으로 변경
```

바로 판단하기 어려운 신고는 `reviewing` 상태로 변경합니다.

```txt
대상 숨김 처리
```

명백한 욕설, 혐오, 개인정보 노출, 도배, 악성 콘텐츠는 숨김 처리합니다.

```txt
신고 기각
```

단순 의견 차이, 반대 의견, 정상적인 비판은 기각합니다.

### 숨김 처리 권장 기준

다음에 해당하면 숨김 처리합니다.

```txt
개인정보 노출
욕설 또는 인신공격
차별 또는 혐오 표현
명백한 허위사실 반복
도배성 글
토론 주제와 무관한 광고
성적 또는 폭력적 콘텐츠
서비스 운영 방해 목적의 글
```

다음은 신중하게 판단합니다.

```txt
강한 반대 의견
풍자
불편하지만 토론 맥락상 필요한 비판
정치적/사회적 논쟁 표현
```

토론 서비스의 목적상, 단순히 의견이 불편하다는 이유만으로 숨김 처리하지 않습니다.

## 7. 공지사항 운영

관리자는 공지사항을 작성, 수정, 삭제할 수 있습니다.

공지 상태:

```txt
draft
published
```

상태 의미:

```txt
draft
```

작성 중인 공지입니다. 일반 유저에게 보이지 않습니다.

```txt
published
```

공개된 공지입니다. 일반 유저가 볼 수 있습니다.

공지사항은 고정 여부를 설정할 수 있습니다.

```txt
is_pinned = true
```

고정 공지는 일반 공지보다 우선적으로 노출됩니다.

### 공지 작성 예시

공지에는 다음 내용을 포함하는 것이 좋습니다.

```txt
변경된 기능
점검 예정 시간
서비스 이용 안내
토론 규칙 안내
테스트 참여 안내
```

## 8. 올리브 가지 기능 운영

올리브 가지 기능은 유저의 재방문을 유도하기 위한 가벼운 참여 기능입니다.

현재 기능:

```txt
하루 한 번 물주기
총 물주기 횟수 기록
연속 물주기 기록
최고 연속 기록 저장
오늘 물주기 여부 표시
```

운영자는 이 기능을 토론 참여 보상, 출석 체크, 커뮤니티 리텐션 기능으로 확장할 수 있습니다.

## 10. 배포 후 확인

GitHub에 push하면 Vercel이 자동 배포를 진행합니다.

운영 주소:

```txt
https://metropolisweb.vercel.app
```

배포 후 Vercel 상태가 `Ready`인지 확인합니다.

문제가 있으면 아래 순서로 확인합니다.

```txt
1. Vercel Build Logs 확인
2. 환경변수 확인
3. Root Directory가 apps/web인지 확인
4. Supabase Auth URL 설정 확인
5. Supabase RLS 정책 확인
```

## 11. Supabase 운영 주의사항

운영 중 직접 SQL을 실행할 때는 주의해야 합니다.

특히 아래 작업은 실행 전 반드시 내용을 검토합니다.

```txt
drop policy
drop function
alter table
delete
update without where
storage policy 변경
```

권장 방식:

```txt
1. SQL을 patches 폴더에 먼저 기록
2. 내용 검토
3. Supabase SQL Editor에서 실행
4. 사이트 기능 테스트
5. Git commit
```

## 12. 장애 대응

### 로그인이 안 되는 경우

확인할 것:

```txt
Supabase Auth URL Configuration
Vercel 환경변수
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

### 주제 목록이 안 보이는 경우

확인할 것:

```txt
topics 상태가 open / active / closed인지
deleted_at이 null인지
topics RLS 정책
```

### 참가가 안 되는 경우

확인할 것:

```txt
topic status가 open인지
join_topic RPC
topic_participants RLS
로그인 상태
profiles.status가 active인지
```

### 글 또는 댓글 작성이 안 되는 경우

확인할 것:

```txt
주제에 참가했는지
topic status가 open 또는 active인지
profiles.status가 active인지
create_debate_post RPC
create_debate_comment RPC
```

### 이미지 업로드가 안 되는 경우

확인할 것:

```txt
debate-images bucket 존재 여부
bucket public 여부
파일 크기 5MB 이하 여부
파일 타입 JPG / PNG / WEBP 여부
storage.objects 정책
이미지 경로 구조
```

### 관리자 페이지 접근이 안 되는 경우

확인할 것:

```txt
profiles.role = admin
profiles.status = active
requireAdmin 서버 함수
로그인 계정이 맞는지
```

## 13. 운영 원칙

Metropolis는 익명 역할 기반 토론 서비스입니다.

운영 시 아래 원칙을 지킵니다.

```txt
1. 유저의 실제 계정 정보를 공개하지 않는다.
2. 의견 차이와 운영 위반을 구분한다.
3. 신고 처리는 일관된 기준으로 한다.
4. 관리 기능은 최소 권한으로 운영한다.
5. DB 변경은 기록을 남긴다.
6. 배포 전 항상 build를 확인한다.
7. 실제 유저 테스트 전 테스트 라우트와 실험 페이지 노출을 막는다.
```
